# Creating Specs

Before implementing a new feature, create a spec file in `specs/`.

## Format

```md
# Feature Name

## Status
[TODO/In Progress/✅ Complete]

## Overview
Brief description of the feature.

## Implementation Details
- Specific technical decisions
- File structure
- Dependencies

## Tasks
- [ ] Task 1
- [ ] Task 2

## References
- Link to official repo/docs
```

## Process

1. Create `specs/feature-name.md`
2. Research and document
3. Implement
4. Update status to ✅
5. Commit with spec
